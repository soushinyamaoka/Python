直近1週間（7日間）のgitコミット履歴から、作業内容を会社報告用に100字程度で簡潔にまとめてください。

## 手順

1. 以下のコマンドで直近1週間のコミット履歴を取得する：
```bash
git log --oneline --since="7 days ago" --no-merges
```

2. 変更内容の詳細を把握するため、差分のstatも確認する：
```bash
git log --stat --since="7 days ago" --no-merges
```

3. 上記の情報をもとに、以下のフォーマットで報告文を作成する：

## 出力フォーマット

```
【レシピ検索API】<作業内容を100字程度で要約>
```

## ルール
- 100字程度に収める（80〜120字の範囲）
- 技術用語は最小限にし、何を実現したかが伝わる表現にする
- 複数の作業があれば箇条書きではなく1文にまとめる
- コミットがない場合は「該当期間のコミットはありません」と報告する
- 報告文はそのままコピペできる形式で出力する
